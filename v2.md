# 舞萌 Bot (koishi-plugin-maibot) v2 功能列表

---

## 一、所有 Bot 指令

### 1.1 帮助与信息

| 指令 | 别名 | 说明 | 权限 | 用法示例 |
|------|------|------|------|----------|
| `/mai` | `/mai帮助` | 查看所有可用指令帮助 | 任意 | `/mai`、`/mai --advanced`（显示高级功能） |
| `/maiping` | - | 测试机台连接 | 任意 | `/maiping` |
| `/maiqueue` | - | 查询当前队列位置 | 任意 | `/maiqueue` |
| `/mai状态 [target]` | `/mymai` | 查询绑定状态/票券/授权信息 | 任意（代查需 auth>=3） | `/mai状态`、`/mai状态 @用户`、`/mai状态 --expired` |
| `/mai查询opt <titleVer>` | - | 查询 Mai2 选项文件下载地址 | 任意 | `/mai查询opt 1.40` |
| `/maialert [on\|off]` | - | 开关个人账号登录/登出播报 | 任意 | `/maialert on`、`/maialert off`、`/maialert`（查状态） |
| `/maialert set <targetUserId> [on\|off]` | - | 设置他人播报状态 | auth>=3 | `/maialert set 123456 on` |

### 1.2 账号绑定管理

| 指令 | 别名 | 说明 | 权限 | 用法示例 |
|------|------|------|------|----------|
| `/mai绑定 [qrCode]` | - | 绑定舞萌 DX 账号（SGID 文本或链接） | 任意 | `/mai绑定`（交互式）、`/mai绑定 https://wq.wahlap.net/qrcode/req/...` |
| `/mai解绑` | - | 解绑舞萌 DX 账号（受换绑冷却限制） | 任意 | `/mai解绑` |
| `/mai解绑卡` | `maiunbindkey` | 冷却期内凭解绑卡额度解绑（需 SGID + 二次确认） | 任意 | `/mai解绑卡`（交互式） |

### 1.3 水鱼 (Fish) B50

| 指令 | 别名 | 说明 | 权限 | 用法示例 |
|------|------|------|------|----------|
| `/mai绑定水鱼 <token> [target]` | - | 绑定水鱼 Token（127-132 字符） | 任意（代绑需 auth>=3） | `/mai绑定水鱼 abc123...` |
| `/mai解绑水鱼 [target]` | - | 解绑水鱼 Token（DX 绑定保留） | 任意（代解需 auth>=3） | `/mai解绑水鱼 @用户` |
| `/mai上传B50 [qrCodeOrTarget]` | `maiu` | 上传 B50 到水鱼 | 任意（代传需 auth>=3） | `/mai上传B50`、`/mai上传B50 SGID...`、`/mai上传B50 @用户` |
| `/maiua [qrCodeOrLxnsCode] [target]` | - | 同时上传 B50 到水鱼和落雪（一次 SGID 流程） | 任意（代传需 auth>=3） | `/maiua`、`/maiua SGID...` |

### 1.4 落雪 (Luoxue) B50

| 指令 | 别名 | 说明 | 权限 | 用法示例 |
|------|------|------|------|----------|
| `/mai绑定落雪 <lxnsCode> [target]` | - | 绑定落雪代码（15 字符） | 任意（代绑需 auth>=3） | `/mai绑定落雪 123456789012345` |
| `/mai解绑落雪 [target]` | - | 解绑落雪代码（DX 绑定保留） | 任意（代解需 auth>=3） | `/mai解绑落雪 @用户` |
| `/mai上传落雪b50 [qrCodeOrLxnsCode] [target]` | `maiul` | 上传 B50 到落雪 | 任意（代传需 auth>=3） | `/mai上传落雪b50`、`/mai上传落雪b50 SGID...` |

### 1.5 票券管理（高级功能）

| 指令 | 别名 | 说明 | 权限 | 用法示例 |
|------|------|------|------|----------|
| `/mai发票 [multiple] [target]` | - | 发放功能票（2-6 倍，默认 2） | 任意（代发需 auth>=3） | `/mai发票`、`/mai发票 4`、`/mai发票 3 @用户` |
| `/mai清票 [target]` | - | 清空所有功能票 | auth>=3 | `/mai清票`、`/mai清票 @用户` |

### 1.6 成绩管理

| 指令 | 别名 | 说明 | 权限 | 用法示例 |
|------|------|------|------|----------|
| `/mai上传乐曲成绩 [target]` | - | 手动上传单曲成绩（交互式） | auth>=3 | `/mai上传乐曲成绩` → 交互输入：musicId, levelId, achievement, combo, sync, dxScore, rank |
| `/mai删除成绩 [target]` | - | 删除指定乐曲成绩（交互式） | auth>=3 | `/mai删除成绩` → 交互输入：musicId, levelId |

### 1.7 收藏品

| 指令 | 别名 | 说明 | 权限 | 用法示例 |
|------|------|------|------|----------|
| `/mai获取收藏品 [qrCodeOrTarget]` | `/mai发收藏品` | 获取收藏品（交互式：类型、ID、数量） | auth>=3 | `/mai获取收藏品`、`/mai获取收藏品 SGID...` |
| `/mai清收藏品 [target]` | - | 清空所有收藏品 | auth>=3 | `/mai清收藏品`、`/mai清收藏品 @用户` |

### 1.8 版本与里程

| 指令 | 说明 | 权限 | 用法示例 |
|------|------|------|----------|
| `/mai修改版本号 [qrCodeOrTarget]` | 修改游戏 ROM/Data 版本号（支持缓存） | auth>=3 | `/mai修改版本号`、`/mai修改版本号 @用户` |
| `/mai舞里程 <mile> [target]` | 发放舞里程（必须是 1000 的倍数；需开启 enableMaimile 配置） | auth>=3 | `/mai舞里程 5000`、`/mai舞里程 3000 @用户` |

### 1.9 锁定与保护

| 指令 | 别名 | 说明 | 权限 | 用法示例 |
|------|------|------|------|----------|
| `/mai锁定 [target]` | - | 锁定账号，防止他人登录 | auth>=3 | `/mai锁定`、`/mai锁定 @用户` |
| `/mai解锁 [target]` | `/mai逃离小黑屋`、`/mai逃离` | 解锁账号（仅限通过 `/mai锁定` 锁定的账号） | auth>=3 | `/mai解锁`、`/mai解锁 @用户` |
| `/mai保护模式 [on\|off] [target]` | - | 开关保护模式（账号离线时自动锁定） | auth>=3 | `/mai保护模式 on`、`/mai保护模式 off @用户` |

### 1.10 卡密与优先授权

| 指令 | 说明 | 权限 | 用法示例 |
|------|------|------|----------|
| `/mai兑换卡密 [code]` | 兑换卡密（个人优先/群组优先/解绑卡） | 任意 | `/mai兑换卡密 MAI-xxx`、`/mai兑换卡密`（交互式） |
| `/mai取消群组优先` | 取消本群群组优先（仅群组卡兑换人可操作） | 任意 | `/mai取消群组优先` |
| `/mai群组优先换绑` | 发起群组优先迁移（在原群执行） | 任意 | `/mai群组优先换绑` |

### 1.11 管理员指令

#### 卡密管理（auth >= authLevelForCardAdmin，默认 4）

| 指令 | 说明 | 用法示例 |
|------|------|----------|
| `/mai管理员生成卡密 [durationSpec] [count]` | 生成卡密 | `/mai管理员生成卡密`（交互式）、`/mai管理员生成卡密 7d 5`、`-g 30d 3`（群组）、`-u -1 10`（解绑） |
| `/mai管理员删除卡密 [code]` | 作废卡密（支持多行/TSV 批量） | `/mai管理员删除卡密`（交互式）、`/mai管理员删除卡密 MAI-xxx` |
| `/mai管理员导出卡密 [scope]` | 导出卡密为 TSV | `/mai管理员导出卡密`（交互式）、`/mai管理员导出卡密 all`、`unused`、`redeemed` |

#### 优先授权管理（auth >= authLevelForCardAdmin，默认 4）

| 指令 | 说明 | 用法示例 |
|------|------|----------|
| `/mai管理员取消群组优先 [guildKey]` | 取消群组优先 | `/mai管理员取消群组优先`、`/mai管理员取消群组优先 xxx` |
| `/mai管理员取消个人优先 <target>` | 清除个人优先 | `/mai管理员取消个人优先 @用户` |
| `/mai管理员设置个人优先 <target> <spec>` | 设置个人优先 | `/mai管理员设置个人优先 @用户 永久`、`7d`、`clear` |
| `/mai管理员设置群组优先 <spec>` | 设置群组优先 | `/mai管理员设置群组优先 30d`、`/mai管理员设置群组优先 30d -g groupId` |
| `/maibypass <target>` | 清除冷却（别名 `/mai管理员清除冷却`） | `/maibypass @用户`、`/maibypass 123456` |

#### 控制与统计（auth >= 3）

| 指令 | 说明 | 用法示例 |
|------|------|----------|
| `/mai管理员关闭所有锁定和保护` | 一键关闭所有锁定和保护模式 | `/mai管理员关闭所有锁定和保护` |
| `/mai管理员关闭登录播报 [state]` | 开关全局登录播报功能 | `/mai管理员关闭登录播报 off` |
| `/mai管理员关闭所有播报` | 强制关闭所有用户的 maialert | `/mai管理员关闭所有播报` |
| `/mai管理员查询操作 <refId>` | 通过 Ref_ID 查询操作详情 | `/mai管理员查询操作 ABC123` |
| `/mai管理员统计` | 查看今日指令执行次数统计 | `/mai管理员统计` |

## 二、所有 API 端点

### 2.1 两种 API 模式

| 模式 | 基础路径 | 认证方式 |
|------|----------|----------|
| **Team（自建）** | `/api/public/*` 或 `/api/private/*` | 无需额外认证（query 参数传 client_id 等） |
| **Public（AWMC 网关）** | `/v1/*` | `Authorization: Bearer <token>` |

### 2.2 公共端点

#### GET `/api/public/mai_ping` 或 `/v1/mai_ping`
**说明**: 测试机台连接

**Team 请求**:
```
GET /api/public/mai_ping
```

**Public 请求**:
```
GET /v1/mai_ping
Authorization: Bearer <token>
```

**响应**:
```json
{
  "returnCode": 0,
  "serverTime": 1717200000000,
  "result": "OK"
}
```

#### POST `/api/private/upload_score_manual` 或 `/v1/upload_score_manual`
**说明**: 手动上传单曲成绩

**请求**:
```
POST /api/private/upload_score_manual
Content-Type: application/json

{
  "qr_code": "<SGID文本>",
  "musicId": 1,
  "levelId": 2,
  "achievement": 1010000,
  "combo": 500,
  "sync": 100,
  "dxScore": 3000,
  "rank": 4,
  "playcount": 1,
  "iscover": 0,
  "isforce": 0,
  "detailmode": 0
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `qr_code` | string | 是 | SGID 二维码文本 |
| `musicId` | number | 是 | 乐曲 ID |
| `levelId` | number | 是 | 难度: 0=Basic, 1=Advanced, 2=Expert, 3=Master, 4=Re:Master |
| `achievement` | number | 是 | 达成率 (如 1010000 = 101.0000%) |
| `combo` | number | 是 | 连击数 |
| `sync` | number | 是 | 同步率 (0-100) |
| `dxScore` | number | 是 | DX 分数 |
| `rank` | number | 是 | 评级: 0=D, 1=C, 2=B, 3=A, 4=AA, 5=AAA, 6=S, 7=S+, 8=SS, 9=SS+, 10=SSS, 11=SSS+ |
| `playcount` | number | 否 | 游玩次数 (默认 1) |
| `iscover` | number | 否 | 是否覆盖 (默认 0) |
| `isforce` | number | 否 | 是否强制 (默认 0) |
| `detailmode` | number | 否 | 详细模式 (默认 0) |

**响应**:
```json
{
  "success": true,
  "result": {
    "returnCode": 0,
    "apiName": "upload_score"
  },
  "msg": "success"
}
```

## 三、数据库表结构

| 表名 | 主键 | 用途 |
|------|------|------|
| `maibot_bindings` | `id`（唯一: `userId`） | 用户绑定（QR 码、maiUid、Token、锁定/保护状态、解绑额度） |
| `maibot_settings` | `key` | 插件全局设置 |
| `maibot_operation_logs` | `id`（唯一: `refId`） | 操作审计日志 |
| `maibot_user_cooldowns` | `userId` + `slot` | 每用户每槽位冷却 |
| `maibot_card_keys` | `id`（唯一: `code`） | 卡密（MAI-xxx...） |
| `maibot_priority_users` | `userId` | 个人优先授权记录 |
| `maibot_group_priority` | `guildKey` | 群组优先授权记录 |
| `maibot_group_rebind_pending` | `anchorUserId` | 待处理的群组优先迁移 |
| `maibot_user_rebind_state` | `userId` | 最后绑定变更时间戳（冷却用） |
| `maibot_user_terms` | `userId` | 条款接受记录 |
